# 🧠 AI RULES - Agent Workflow (For Care_AI Project)

> Mục tiêu: Điều khiển AI hiểu đúng project Care_AI, code đúng context, tránh phá structure, giảm bug và tech-debt.

---

# 🔥 1. CORE RULE (LUÔN DÙNG)

## Prompt
Act as a senior fullstack developer working on Care_AI.

Understand the project structure before coding.

Project modules:
- backend → API, database, business logic
- web_admin → React admin dashboard
- mobile_app → mobile application
- ai_service → AI processing
- skills → reusable logic

Rules:
- Only work on provided code
- Do not scan entire project unless required
- Do not modify unrelated files
- Reuse existing logic whenever possible
- Keep code minimal, clean, and consistent

If anything is unclear, ask before implementing.

## 📝 Giải thích
- Đây là **não chính của AI**
- Giúp AI hiểu project multi-module của m  
- Không cho AI code lung tung hoặc phá structure  
- Tập trung vào:
  - minimal change  
  - reuse code  

---

# 🚀 2. IMPLEMENTATION RULE (KHI APPLY CODE)

## Prompt
Analyze the task based on current module.

Steps:
1. Understand context (file + module)
2. Identify problem or requirement
3. List affected files
4. Propose minimal solution
5. Then implement

Ensure:
- follow existing patterns
- keep logic consistent
- do not over-engineer

Do not:
- skip steps
- implement outside scope

## 📝 Giải thích
- Ép AI suy nghĩ như dev thật  
- Không nhảy vào code ngay  
- Giảm lỗi sai logic + code bừa  

---

# 🔒 3. SCOPE CONTROL RULE (QUAN TRỌNG)

## Prompt
Before coding:

- List all files that will be modified
- Explain why each file is affected

Only modify:
- files directly related to the task

If unsure → ask before proceeding.

## 📝 Giải thích
- Rule quan trọng nhất để **tránh phá project**  
- Giúp AI không sửa lan sang module khác  
- Rất cần với project nhiều folder như m  

---

# 🔁 4. REUSE RULE

## Prompt
Check existing code before writing new code.

Prefer:
- reuse existing functions/components
- extend instead of rewrite

Avoid:
- duplicate logic
- creating new patterns unnecessarily

## 📝 Giải thích
- Giữ code đồng nhất  
- Tránh mỗi chỗ 1 kiểu  
- Giảm tech-debt  

---

# 🔧 5. BACKEND RULE (API + DB)

## Prompt
When working on backend:

Ensure:
- do not break existing API responses
- keep database consistency
- follow existing service/controller structure

Check:
- authentication flow
- RBAC (if applicable)

## 📝 Giải thích
- Backend là phần dễ gây lỗi nhất  
- Rule này giúp:
  - không vỡ API  
  - không lỗi frontend  

---
# 🎨 6. FRONTEND RULE (web_admin - HTML)

## Prompt
When working on frontend:

Ensure:
- keep pure HTML/CSS/JS structure
- maintain current layout and styling
- use existing classes and structure

Do not:
- convert to React
- introduce frameworks
- rewrite entire page

If updating UI:
- modify only necessary parts
- keep consistent styling

## 📝 Giải thích
- Project hiện tại dùng HTML thuần  
- Không cho AI tự chuyển sang React  
- Tránh phá layout  
- Chỉ sửa đúng phần cần thiết  

# 📱 7. MOBILE RULE (mobile_app)

## Prompt
When working on mobile_app:

Ensure:
- do not break navigation
- keep UI responsive
- follow existing screen structure

## 📝 Giải thích
- Mobile dễ lỗi navigation  
- Rule này giữ app ổn định  

---

# 🤖 8. AI SERVICE RULE

## Prompt
When working on ai_service:

Ensure:
- logic is correct and predictable
- do not change model behavior unless required
- keep functions modular

## 📝 Giải thích
- Tránh AI chạy sai logic  
- Giữ code AI ổn định

# 🔍 9. VERIFICATION RULE (SAU KHI CODE)

## Prompt
Review implementation.

Check:
- does it break existing features?
- is logic correct?
- is it consistent with current codebase?

List all issues clearly.

## 📝 Giải thích
- AI tự review lại code  
- Phát hiện lỗi trước khi chạy thật  
- Giảm bug rất mạnh  

---

# 🔧 10. FIX RULE (KHI CÓ BUG)

## Prompt
Fix the issue.

Steps:
1. Identify root cause
2. Apply minimal fix
3. Ensure no side effects

Explain:
- what was wrong
- what was fixed

## 📝 Giải thích
- Không fix bừa  
- Fix đúng gốc lỗi  
- Không phá feature khác  

---

# 🎨 11. UI FIX RULE (CHỈ SỬA GIAO DIỆN)

## Prompt
Fix UI issues without changing business logic.

Ensure:
- consistent spacing
- consistent typography
- consistent components

## 📝 Giải thích
- Chỉ sửa UI  
- Không đụng logic  
- Dùng khi UI bị lệch  

---

# 🧪 12. VERIFY AFTER FIX

## Prompt
Verify that the fix does not introduce regressions.

## 📝 Giải thích
- Check lại toàn hệ thống sau khi fix  
- Tránh lỗi dây chuyền  

---

# 🧠 WORKFLOW CHUẨN (QUAN TRỌNG)

## Flow

1. Apply CORE RULE + IMPLEMENTATION RULE  

2. AI:
   - hiểu context  
   - list affected files  

3. AI implement minimal solution  

4. Dùng VERIFICATION RULE  

5. Nếu có lỗi:
   - dùng FIX RULE
